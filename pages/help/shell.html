Unix Shells Comparison

Understanding the differences between csh, bash, bsh and Korn shell

Bash

Bourne Again SHell - The default shell on most Linux distributions

  • Release: 1989
  • Creator: Brian Fox
  • Features: Command history, job control, shell functions
  • Scripting: Extensive scripting capabilities
  • Compatibility: POSIX compliant, backward compatible with sh
# Example: For loop
for i in {1..5}; do
  echo "Number: $i"
done

Bsh / sh

Bourne SHell - The original Unix shell

  • Release: 1977
  • Creator: Stephen Bourne
  • Features: Basic shell functionality
  • Scripting: Limited compared to modern shells
  • Compatibility: Foundation for POSIX shell standard
# Example: Variable assignment
name="John"
echo "Hello, $name"

Csh

C SHell - Shell with C-like syntax

  • Release: 1978
  • Creator: Bill Joy
  • Features: Command history, job control, aliases
  • Scripting: C-like syntax, but limited for scripting
  • Compatibility: Not POSIX compliant
# Example: If statement
if ($user == "john") then
  echo "Hello John"
endif

Korn Shell

KSH - Advanced shell with extensive features

  • Release: 1983
  • Creator: David Korn
  • Features: Command-line editing, associative arrays
  • Scripting: Advanced scripting capabilities
  • Compatibility: POSIX compliant, backward compatible with sh
# Example: Associative array
typeset -A person
person[name]="John"
person[age]=30
echo ${person[name]}

Evolution of Unix Shells

1971

Thompson shell - The first Unix shell

1977

Bourne shell (sh) released - Became the standard Unix shell

1978

C shell (csh) released - Introduced C-like syntax

1983

Korn shell (ksh) released - Added advanced scripting features

1989

Bash released - Became the default shell for most Linux distributions
